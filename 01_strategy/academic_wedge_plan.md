# Wedge académique — Opays Tech

Date: 2026-05-28  
Objet: transformer l’intuition “universités / recherche” en première verticale de démonstration sans perturber Forge / Sovereign.

## 1) Pourquoi ce wedge est pertinent

Le secteur académique a plusieurs caractéristiques utiles :

- besoin récurrent de recherche et de structuration ;
- beaucoup de travail documentaire ;
- présence de processus répétitifs ;
- valeur du temps bien visible ;
- contexte favorable aux assistants IA et à la base de connaissances ;
- besoin local de solutions fiables, simples et contextualisées.

Ce wedge est donc intéressant parce qu’il combine :
- **douleur réelle**,
- **usage répétitif**,
- **valeur défendable**,
- **preuve rapide**.

---

## 2) Ce que nous vendons vraiment

Nous ne vendons pas “une IA”.
Nous vendons un **assistant de recherche et de workflow** qui aide les utilisateurs à :

- trouver et organiser l’information ;
- suivre les projets et les tâches ;
- structurer les travaux et les livrables ;
- réduire les pertes de temps liées aux documents ;
- conserver une mémoire exploitable ;
- travailler dans un cadre multilingue et contextuel.

---

## 3) Cible prioritaire

La cible de départ n’est pas “tous les étudiants”.
La cible doit être précise :

- universités et facultés ;
- laboratoires et équipes de recherche ;
- directions académiques ;
- encadreurs de mémoire ;
- bureaux d’innovation / transformation numérique ;
- écoles supérieures et instituts.

---

## 4) Cas d’usage prioritaires

### A. Assistant de recherche
- aider à retrouver des références ;
- résumer des documents ;
- classer les sources ;
- structurer une problématique ;
- générer une première trame de travail.

### B. Workflow documentaire
- suivre l’état d’un projet ;
- centraliser les versions ;
- limiter les pertes d’information ;
- garder une traçabilité.

### C. Base de connaissances
- capitaliser les travaux précédents ;
- éviter de repartir de zéro ;
- rendre les connaissances réutilisables.

### D. Assistant contextuel
- répondre dans la langue et le niveau de détail utile ;
- guider les utilisateurs dans leurs démarches ;
- servir d’interface de navigation dans l’information.

---

## 5) Proposition de produit / service

### Option 1 — Audit / diagnostic
Un audit d’efficience académique pour identifier :
- les frictions de recherche ;
- les pertes de temps documentaires ;
- les besoins d’organisation ;
- les opportunités d’automatisation.

### Option 2 — Sprint de mise en place
Un sprint court pour :
- installer un premier flux ;
- structurer une base documentaire ;
- mettre en place un assistant limité mais utile ;
- valider le fonctionnement réel.

### Option 3 — Retainer / accompagnement
Un suivi mensuel pour :
- améliorer le système ;
- ajouter de nouveaux cas d’usage ;
- maintenir les connaissances ;
- accompagner l’équipe dans l’adoption.

---

## 6) Produit minimum viable recommandé

Le premier MVP ne doit pas chercher à tout faire.

Il devrait contenir :
- ingestion de documents autorisés ;
- recherche assistée ;
- synthèse des contenus ;
- classification simple ;
- réponse conversationnelle contrôlée ;
- interface très simple ;
- journal des actions.

### À éviter au départ
- entraînement lourd de modèle ;
- fonctions trop larges ;
- promesse de remplacement total du travail humain ;
- complexité d’interface inutile.

---

## 7) Modèle économique possible

### Modèle de départ
- audit / diagnostic initial ;
- sprint d’implémentation ;
- retainer mensuel.

### Logique de prix
Le prix doit être lié :
- au nombre d’utilisateurs ;
- au volume documentaire ;
- au niveau d’accompagnement ;
- à la sensibilité des données ;
- à la rapidité d’exécution demandée.

### Objectif de validation
Valider :
- le besoin réel ;
- la disposition à payer ;
- la rétention ;
- la facilité de déploiement.

## 9) Comment ce wedge s’insère dans Opays

Le wedge académique ne remplace pas Forge / Sovereign.

Il sert de :
- **preuve de marché** ;
- **terrain d’apprentissage** ;
- **verticale démonstrative** ;
- **base de contenu et de crédibilité** ;
- **source potentielle de revenus récurrents**.

Forge peut livrer.
Sovereign peut approfondir.
Le wedge académique montre que la méthode fonctionne dans un contexte réel.
