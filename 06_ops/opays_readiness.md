# Opays Tech - Readiness Operatoire

Ce document resume ce qui doit etre en place pour considerer l'environnement Opays comme pret a executer, vendre et livrer.

## 1. Ce qui est deja en place

- Base strategique centralisee dans `OPAYS_HQ.md`
- Regles de travail dans `CLAUDE.md` et `AGENTS.md`
- Carte du workspace dans `WORKSPACE_MAP.md`
- Gouvernance tracee dans `05_decisions/decision_log.md`
- Compass branche au registre documentaire
- CLI Opays pour verifier agents, competence et progression
- Skills Codex et Anthropic installes pour:
  - code / review / refactor
  - frontend / brand / design
  - documents / presentations / tables

## 2. Ce qui doit rester vrai

1. Une seule source de verite pour la vision et la gouvernance.
2. Des changements petits, traçables et reversibles.
3. Une offre lisible avant tout enrichissement technique.
4. Une interface simple, claire et credible.
5. Des agents utilises comme amplificateurs, pas comme decoration.

## 3. Ce qu'il faut produire pour etre vraiment pret

- une offre phares explicite par branche:
  - Forge
  - Sovereign
- un kit de livraison repetable:
  - audit
  - proposition
  - execution
  - verification
- des templates reutilisables:
  - diagnostic
  - roadmap
  - compte rendu
  - proposition
- des indicateurs simples:
  - temps gagne
  - delai
  - clarté obtenue
  - valeur livree

## 4. Utilisation quotidienne conseillee

- `npm run opays:agents`
- `npm run opays:compass`
- `npm run opays:compliance`
- `npm run opays:progress`
- `npm run opays:readiness`

## 5. Statut cible

L'environnement est consideré pret lorsque:

- la gouvernance est claire,
- les skills utiles sont installes,
- le CLI de pilotage est accessible,
- les offres sont lisibles,
- la livraison peut etre repetee sans improvisation.
